# Rule: Documentation Integrity

## Constraint
Every turn that involves non-trivial discovery of system logic, hidden paths, or complex event flows MUST lead to a corresponding update in the project documentation (Worklog or Architecture docs).

## Requirements
- Never let a finding go undocumented.
- Discovered knowledge must be captured before the turn ends.
- Prefer `.agents/tickets/4-done/_archive-worklog.md` for session-based discovery logs.
- Prefer `.agents/architecture/` for structural or foundational changes.
